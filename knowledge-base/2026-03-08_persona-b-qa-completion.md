## 全20問の一覧

### カテゴリ1: 技術選定（5問）

| # | 質問 | 回答の要点 |
|---|------|-----------|
| Q1 | LightGBMとXGBoost、どちらを選ぶべきか? | データ規模で判断。大規模→LightGBM、中小規模→XGBoost。両方ベンチマークを推奨 |
| Q2 | RAG構築のベストプラクティスは? | 4ステップ: ドキュメント準備→チャンク分割(200-500トークン)→ベクトルDB選定→検索精度評価 |
| Q3 | 自社サービスに生成AIを組み込む際の技術選定基準は? | API vs セルフホスト、モデル選択、コスト×レイテンシの3軸で判断 |
| Q4 | MLOpsツールは何を使うべきか? | 小規模→MLflow、中規模→W&B、大規模→Kubeflow。まずMLflowから |
| Q5 | ベクトルDBは何を選ぶべきか? | 小規模→Chroma、中規模→Pinecone、大規模→Milvus/Qdrant |

| Q10 | 本番障害が起きたときの対応フローは? | 検知→トリアージ→対処→復旧→事後分析の5ステップ。障害対応プレイブック作成 |

## 主な知見

### 回答設計のパターン
- **結論ファースト**: CTOは忙しいため、最初の2-3行で要点を掴めるようにする
- **選択基準の明確化**: 「小規模なら○○、大規模なら△△」と判断軸を示す
- **具体的な数値**: コスト、期間、精度向上率など定量的な情報を含める
- **実装事例の挿入**: SaaS企業200名規模の事例で共感を得る
- **次のアクション**: 「明日からできること」を1つ提示

### ペルソナBの特徴的なニーズ
- Web開発は得意だがML/AIは専門外 → 技術選定の「正解」を知りたい
- 採用が決まらない → 既存メンバーの育成と外部活用の具体策
- PoCは成功するが本番化で失敗 → 本番化のチェックリストと段階的アプローチ
- 社内に壁打ち相手がいない → 技術顧問の具体的な価値を示す
