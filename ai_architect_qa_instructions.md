# 生成AI変革アーキテクト Q&A資料作成指示書

## 概要

本指示書は、生成AI変革アーキテクト（コンサルタント）の想定質問と回答集をWord形式で作成するためのものです。3つの顧客ペルソナごとに、彼らが本当に聞きたい質問と、期待を超える回答を作成します。

---

## 出力ファイル仕様

### ファイル形式
- 形式: .docx（Microsoft Word）
- ファイル名: `AI_Architect_QA_Guide.docx`
- 出力先: `/mnt/user-data/outputs/`

### ページ設定
- 用紙サイズ: A4
- 余白: 上下左右 2.5cm（約1,000 DXA）
- フォント: 游ゴシック（本文）、游ゴシック Bold（見出し）
- 本文サイズ: 11pt
- ページ番号: フッターに配置（全ページ）

### 構成
1. 表紙（1ページ）
2. 目次（1ページ）
3. ペルソナA: CFO・経営層向けQ&A（約20問 × 各1ページ）
4. ペルソナB: CTO・技術責任者向けQ&A（約20問 × 各1ページ）
5. ペルソナC: 新規事業責任者向けQ&A（約20問 × 各1ページ）

## 回答作成ガイドライン

### 回答の基本構成（1問1ページに収める）

```
【質問】
（質問文をそのまま記載）

【回答】

■ 結論（2-3行）
- まず結論を端的に述べる
- 相手のペインに寄り添う一言を添える

■ 背景・分析（3-5行）
- なぜその問題が起きているのか
- 業界全体の傾向や他社の状況

■ 解決アプローチ（本文の核心、10-15行）
- 具体的なアクションステップ（3-5項目）
- 各ステップの実行イメージ
- 可能であれば事例を1つ挿入

■ 期待される成果（2-3行）
- 何が改善されるか
- 定量的な目安（あれば）

■ 次のアクション（1-2行）
- 今すぐできる具体的な一歩
```

### 回答のトーン・スタイル

1. **一流のプレゼンターとして**
   - 自信を持って語るが、傲慢にならない
   - 専門用語は使うが、必ず平易な説明を添える
   - 「〜と考えます」「〜をお勧めします」の丁寧な表現

2. **ペルソナの課題に寄り添う**
   - 「その悩み、多くの企業様で見てきました」
   - 「ご不安はよく分かります」
   - 共感 → 解決策 → 励ましの流れ

3. **論理的かつ具体的**
   - 抽象論で終わらせない
   - 「例えば〇〇社では...」「私のクライアントでは...」
   - 数字で示せるものは数字で示す

4. **事例の挿入**
   - 各回答に1つは具体的事例を含める
   - 事例は以下から適宜引用・アレンジ：
     - Amazon累計7億円売上創出
     - 大手5社で各3,000万円超の増益実績
     - 半年で年商5,000万円のAI事業立ち上げ
     - ROAS 30%改善
     - PoC成功率を3倍に向上させた経験

5. **アクションを明確に**
   - 「まず明日からできることは〇〇です」
   - 「週1の壁打ちで、〇〇が変わります」

### Step 3: 各ペルソナのQ&Aを作成

**重要：回答作成時のAIエージェント機能発揮**

各質問に対して、以下のプロセスを実行：

1. **分析**: 質問の本質的な課題を特定
2. **仮説立案**: 想定される原因と解決策を複数検討
3. **リサーチ**: Web検索で最新事例・データを収集
4. **回答作成**: 上記ガイドラインに沿って回答を構成

### Step 4: Word出力

```javascript
Packer.toBuffer(doc).then(buffer => {
  fs.writeFileSync('/mnt/user-data/outputs/AI_Architect_QA_Guide.docx', buffer);
  console.log('Document created successfully!');
});
```
